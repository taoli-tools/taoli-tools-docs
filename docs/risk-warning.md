# 风险提示

## 安全风险

除了 Telegram 用户信息和登陆信息之外，所有的数据均保存在你本机的浏览器中，没有加密也无法加密。

因此你更换浏览器后，你的 API Key/Secret，添加的交易对，等等，将不会同步，需要重新添加。

清理浏览器缓存后，这些信息将会消失，需要重新添加。

你电脑上恶意的浏览器插件或木马程序可能会偷走你的 API Key/Secret。

请将 API Key/Secret 设置为所需的最小权限（只给读取和交易权限，千万不要给转账提现权限），并使用安全的操作系统和浏览器环境，以保护资金安全。

建议限制 API Key/Secret 的 IP 白名单。但如果黑客控制了你的电脑，IP 白名单也无法保护你。

如果程序源代码遭受了供应链攻击，你的 API Key/Secret 也可能会被偷走。

如果你在云服务器上运行此程序，你的服务器被黑，你的 API Key/Secret 也会被偷走。

如果 API Key/Secret 被盗，即使开启了 IP 白名单，即使没有转账提现权限，黑客仍然可以通过小币种对敲的方式转移资金。可以通过币种白名单模式限制一下，但用处不大，而且很不方便。

## 金融风险

由于网络不稳定，或者软件 bug 等原因，可能出现只有单边成交的情况，遇到这种情况需要你手动到交易所里操作摆平。

由于币价剧烈波动，永续合约可能会出现爆仓的情况，请注意控制杠杆倍数并及时追加保证金（也是手动在交易所里操作）。

你的仓位可能会触发交易所的 ADL。

系统假设 1 USD = 1 USD*稳定币，当稳定币脱锚时可能会打破原有的假设，会承受到脱锚的风险。

极端行情下，你可能会损失全部本金。

## 交易界面显示的风险信息（Risks）

目前总共会显示三种风险信息：

- ADL（自动减仓优先级）
    - 显示为下图中的五个等级，从左到右被 ADL 的概率依次升高
    
    ![image.png](risk-warning/image.png)
    
    - 目前支持 ADL 显示的交易所：Binance、Bitget、Bybit、Gate、OKX
- MR（保证金率）
    - 显示为一个数字，大于 100 时不显示。
    - 各个交易所的保证金率计算和显示方式不一定相同，这里统一采用 Binance 的显示方式：权益 / 保证金，即数字越大风险越低。至于权益和保证金具体是如何计算的，由各个交易所自己决定，工具只拿到交易所返回的数字并简单后处理一下（通常是取一次倒数）。
- Liq Price（清算价格）
    - 显示为一个$美元符号开头的数字。
    - 爆仓价百分比的计算方式为：`市价 = ( 买一价 + 卖一价 ) / 2`
    - 其中`爆仓价百分比 = 爆仓价 > 市价 ? 爆仓价 / 市价 : 市价 / 爆仓价`

用颜色划分了危险等级，从危险到安全分别为：红色、黄色、绿色、灰色。

不同交易所提供的风险信息种类不同，有数据就显示，没数据就不显示，但不一定代表没有风险。

三种风险信息按顺序显示，排在后面的可能显示不下，会显示…。
